# Building Causal Explanations

Use this reference when constructing or critiquing a causal explanation.

## Minimal Standard

A causal explanation should include:

```text
Under scope condition S, cause X changes mechanism M, which produces outcome Y.
This claim is stronger than rival explanations R because evidence O should appear if M operates and should not appear, or should appear differently, if R is true.
```

## Four Core Requirements

### 1. Temporal Order

The cause must precede the outcome. If X occurs after Y, X cannot explain Y unless the claim is about anticipation, expectation, or feedback and the mechanism explicitly states that.

Check:

- Did X appear before Y?
- Did changes in X occur before changes in Y?
- Is there a lag structure?
- Could Y have caused X?
- Could X and Y be simultaneous effects of another factor?

### 2. Correlation Or Relevant Association

X and Y should show relevant co-variation or patterned connection. Association can be quantitative, comparative, or qualitative.

Forms:

- Presence of X is associated with presence of Y.
- Increase in X is associated with increase/decrease in Y.
- Cases with X differ from comparable cases without X.
- Within a case, changes in X precede changes in Y.
- Process evidence shows a patterned chain from X to Y.

Association is necessary for many claims but not sufficient for causation.

### 3. Excluding Rival Explanations

A causal explanation must consider alternative causes. It should not assume that the favored cause is the only plausible cause.

Common IR rival families:

- Systemic/structural: polarity, distribution of power, security dilemma, balance of threat.
- Domestic: regime type, elite coalition, public opinion, interest groups, bureaucratic politics.
- Economic: trade dependence, sanctions, resource constraints, development level.
- Ideational: identity, ideology, norms, threat perception, historical memory.
- Institutional: alliance design, treaty rules, monitoring, enforcement, veto points.
- Leadership: beliefs, risk propensity, learning, personal networks.
- Historical: path dependence, prior war, colonial legacy, critical juncture.

Ask:

- What else could cause Y?
- Does the rival cause precede both X and Y?
- Is the rival actually a confounder, mediator, or scope condition?
- What evidence would distinguish the main explanation from the rival?

### 4. Counterfactual Or Mechanism

After temporal order and association, strengthen causal inference by at least one of:

- Counterfactual reasoning: if X had been absent/different, Y would have been absent/different.
- Mechanism tracing: X generated Y through identifiable steps.

The strongest designs often use both.

## Counterfactual Logic

A counterfactual should be plausible, minimally rewritten, and theoretically disciplined.

Good counterfactual:

```text
If X had not occurred while background conditions remained similar, Y would probably not have occurred, because the mechanism M would not have been triggered.
```

Weak counterfactual:

```text
If history had been completely different, Y would not have happened.
```

Check:

- What exact value of X changes?
- Which background conditions are held constant?
- Why is the alternative plausible?
- What evidence supports this imagined comparison?
- What comparable case approximates the counterfactual?

## Mechanism Logic

A mechanism explains production, not just association. It should connect cause and outcome through a sequence.

Mechanism components:

- Entities: states, leaders, bureaucracies, alliances, firms, publics, international organizations.
- Activities: threaten, bargain, signal, mobilize, sanction, learn, comply, defect, coordinate.
- Resources: military capability, legitimacy, information, money, institutional authority.
- Beliefs: threat perception, identity, reputation, ideology, expectation.
- Sequence: step 1 produces step 2, step 2 produces step 3, and so on.

Template:

```text
X changes actor A's belief/incentive/capability/information.
A then chooses behavior B.
B changes interaction or institutional condition C.
C triggers response D by actor E.
The accumulated process produces outcome Y.
```

## Explanation Strength

A strong causal explanation is:

- Clear: X, Y, M, S, and R are explicit.
- Directional: arrows and time order are coherent.
- Generative: the mechanism can produce the outcome.
- Discriminating: it yields evidence that differs from rival explanations.
- Bounded: it states where and when it applies.
- Testable: observable implications can be checked.
- Revisable: it can be modified when evidence fails.

## Common Errors

- Treating a label as a cause: “national interest,” “culture,” or “power” must be specified.
- Treating chronology as mechanism: “then this happened” is not enough.
- Treating correlation as proof.
- Ignoring reverse causality.
- Using too many controls without theory.
- Controlling for mediators.
- Selecting only successful cases.
- Making a universal claim from one case.
- Explaining the outcome with facts discovered after the outcome.
